01 AI Agent 开发 本地探测 + 知识增强

NetDiag Agent 本地网络诊断智能体

2026.05 - 至今

面向个人网络排障场景，把分散的本地诊断命令整合为可交互、可解释的网络诊断 Agent。

Python Streamlit LangGraph DeepSeek API ChromaDB RAG
查看项目仓库

项目背景

用户往往只能描述“网页很慢”或“某个网站打不开”，但问题背后可能涉及 DNS、网关、丢包或路径异常，手动排查门槛高。

关键实现

  • 基于 LangGraph 搭建工具调用式诊断流程，接入大模型做诊断规划与结果解释。
  • 封装网关、DNS、延迟、丢包、路由路径等多维检测工具，形成真实探测闭环。
  • 加入向量知识库、长期记忆与轻量 Query 改写，提升口语化问题下的诊断质量。

结果与价值

实现真实探测、知识检索与记忆增强结合的诊断原型，提升个人网络问题定位效率与结果可解释性。

Python FastAPI Streamlit LangGraph Hybrid RAG SQLite Docker

项目背景

传统资料查询依赖人工翻阅文档，结果难溯源、追问体验弱，且不同资料来源之间缺少统一入口。

关键实现

  • 完成文档上传解析、切分、索引与问答全链路，实现资料入库到前端交互闭环。
  • 构建关键词检索 + 向量检索 + RRF 融合的 Hybrid RAG，兼顾召回与相关性。
  • 基于 LangGraph 设计 KB-first Agent 路由，并实现上下文感知 Query Rewrite 与会话记忆。

结果与价值

把人工翻阅手册的流程收敛成可检索、可引用、可追问的知识问答体验，提升资料定位效率与回答可信度。
03 自动化工作流 采集 - 蒸馏 - 沉淀

AI 行业情报自动采集与知识蒸馏系统

2026.06 - 至今

围绕 AI 行业资讯跟踪，搭建“自动采集 - 周级蒸馏 - 长期沉淀”的知识闭环系统。

Obsidian Codex PowerShell Scheduled Tasks Markdown GitHub

项目背景

AI 行业更新快、来源分散、长期知识难沉淀，手工整理既耗时，也很难形成稳定的知识迭代机制。

关键实现

  • 搭建约 20 个资讯入口的采集链路，覆盖 GitHub、RSS、Hacker News、Reddit 等来源。
  • 通过 Codex 自动执行日报整理、周报蒸馏与长期知识回填，降低重复整理成本。
  • 结合 PowerShell 脚本与任务调度，完成发布、同步与本地知识库维护。

结果与价值

打通日报采集、自动整理、周报生成与长期回填链路，预计每天节省 30 - 45 分钟整理时间。
